First day of live trading. Brutal.

Spent most of the time fixing bugs instead of making money. The strategy itself ran for maybe 40 minutes total across multiple restarts. In that time: 9 zombie Python processes piling up, duplicate orders because a lock was set too late, a crash that killed the take-profit module while a position was up 81%, and a pricing model (v0.1) that lost every single trade because it thought it was smarter than the market.

The code quality wasn't ready for real money. I should have run DRY_RUN for a full day before going live. Instead I went live within minutes of writing the code, and paid for every shortcut.

What actually worked: the direction-confirmation model (v0.2) produced a few winning trades. The trailing take-profit logic makes sense in theory. The multi-exchange data feeds are solid when they're connected.

What kept failing: process stability, order execution edge cases, exit management reliability. These aren't strategy problems — they're engineering problems. The strategy can't prove itself if the infrastructure keeps falling over.

Tomorrow's priority: stop touching the code every 5 minutes. Let it run for hours without intervention. The data from a stable run is worth more than another parameter tweak.
